# Microsoft Teams Reporting (Final Step)

The final step of the pipeline delivers the AI-supported report to a specified
Microsoft Teams channel using an Incoming Webhook and an Adaptive Card.

## What Gets Sent

- The Nova-generated summary (rendered as bullet points)
- Three chart images:
  - ML Score Distribution
  - URL Length Distribution
  - Top Suspicious Keywords

Because Microsoft Teams cannot render local files, the charts are first
uploaded to S3 and referenced in the Adaptive Card via presigned URLs.

## Prerequisites

1. A Microsoft Teams **Incoming Webhook** URL for the target channel.
2. AWS credentials with permission to:
   - `s3:PutObject` and `s3:GetObject` on the chart-hosting bucket/prefix.
3. The three chart images already generated by `src/ai_analysis_report.py`.

## Create a Teams Incoming Webhook

1. In Microsoft Teams, open the target channel.
2. Select **... (More options)** -> **Connectors** (or **Workflows** for the
   newer Workflows-based webhooks).
3. Add/configure an **Incoming Webhook**.
4. Name it (for example `URL Model Report`) and create it.
5. Copy the generated webhook URL.

## Run the Final Step Standalone

First save the summary text to a file (for example `summary.txt`), then:

```bash
python src/send_teams_report.py \
  --webhook-url "https://prod-XXX.logic.azure.com:443/workflows/.../invoke?..." \
  --summary-file "summary.txt" \
  --bucket "example-bucket" \
  --prefix "mlmodels/urlmodel/reports" \
  --chart-score "chart_score.png" \
  --chart-length "chart_length.png" \
  --chart-keywords "chart_keywords.png" \
  --title "URL Model Scan Report (AI-supported)"
```

## Run as Part of the Analysis Step

`src/ai_analysis_report.py` can deliver to Teams directly after generating the
charts and summary, by passing `--teams-webhook`:

```bash
python src/ai_analysis_report.py \
  --csv-file "VT_20250301_63.csv" \
  --region "eu-west-2" \
  --nova-model-id "amazon.nova-lite-v1:0" \
  --output-dir "." \
  --teams-webhook "https://prod-XXX.logic.azure.com:443/workflows/.../invoke?..."
```

## Payload Format

The module posts an Adaptive Card (`application/vnd.microsoft.card.adaptive`):

```json
{
  "type": "message",
  "attachments": [
    {
      "contentType": "application/vnd.microsoft.card.adaptive",
      "content": {
        "$schema": "http://adaptivecards.io/schemas/adaptive-card.json",
        "type": "AdaptiveCard",
        "version": "1.4",
        "body": [
          { "type": "TextBlock", "text": "URL Model Scan Report (AI-supported)" },
          { "type": "TextBlock", "text": "- Total URLs Monitored: 7" },
          { "type": "Image", "url": "https://...presigned-chart..." }
        ]
      }
    }
  ]
}
```

## Troubleshooting

- `400`/`401` from webhook: the webhook URL is invalid or expired; recreate it.
- Images not displaying: presigned URL expired (default 1 hour) or the S3
  object is not readable; re-run so a fresh presigned URL is generated.
- `AccessDenied` on upload: check `s3:PutObject` permission on the bucket/prefix.
